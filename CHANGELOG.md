# Changelog

All notable changes to `sqlc-cobra` are documented in this file.
Format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).
The project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [v0.1.0] — 2026-05-11

Initial release. Clean extraction of a production MySQL implementation.

### Requirements

- Go **1.24** or newer (dictated by the transitive `mysql` and
  `filippo.io/edwards25519` dependencies).

### Added

**`runtime/` package** (`github.com/sat0yu/sqlc-cobra/runtime`)

| Symbol | Description |
|---|---|
| `Stdin`, `Stdout`, `Stderr` | Package-level `io.Reader` / `io.Writer` vars defaulting to `os.Stdin` / `os.Stdout` / `os.Stderr`. Tests can reassign them to capture or inject content without spawning a subprocess. |
| `ErrAborted` | Sentinel error returned when the user declines a mutation prompt |
| `IsAborted(err)` | Unwrap-aware test for `ErrAborted` |
| `Prompt{In, Out}` | Mutation confirmation flow with injectable IO |
| `Prompt.ConfirmMutation(cmd, sql, args)` | Prints SQL preview + `y/N` prompt; `--yes` flag bypasses |
| `PrintInt64(w, n)` | Writes an int64 followed by newline |
| `PrintOK(w)` | Writes `"ok"` followed by newline |
| `PrintJSON(w, v)` | Marshals with `sql.Null*` flattening and 2-space indent |
| `PrintExecResult(w, r)` | Formats `sql.Result` as `{"lastInsertId":…,"rowsAffected":…}` |
| `CmdContext(cmd)` | Returns `cmd.Context()` or `context.Background()` as fallback |
| `ParseInt32(s)` | Parses decimal string to `int32` |
| `ParseInt64(s)` | Parses decimal string to `int64` |
| `ParseInt(s)` | Parses decimal string to `int` |
| `ParseBool(s)` | Parses boolean string |
| `ParseTime(s)` | Parses RFC3339 string to `time.Time` |
| `NullStringFromFlag(cmd, flag)` | Returns `sql.NullString`; `Valid=false` if flag was not set on the CLI |

**`cmd/sqlc-cobra-gen` binary**

Flags:

| Flag | Default | Required? |
|---|---|---|
| `-src` | `internal/db/queries` | — |
| `-out` | `zz_generated_commands.go` | — |
| `-pkg` | `sqlc` | — |
| `-queries-import` | *(none)* | **yes** |
| `-parent` | `SqlcCmd` | — |
| `-adapter` | `openQueries` | — |
| `-runtime-import` | `github.com/sat0yu/sqlc-cobra/runtime` | — |
| `-mutating-prefixes` | `Create,Update,Delete,Insert,Prune` | — |
| `-engine` | `mysql` | — |

**Generated file contract**

The generated `zz_generated_commands.go` file:

- Carries a `// Code generated by sqlc-cobra v0.1.0` header.
- Uses `runtime.CmdContext`, `runtime.PrintXxx`, `runtime.Prompt`, and
  `runtime.Stdin` / `runtime.Stdout` / `runtime.Stderr` from the runtime
  package — no `os.Std*` references.
- Calls the consumer-supplied `openQueries()` adapter for the DB handle.
- Registers every command with `<Parent>.AddCommand(cmd)` in `init()`.

**`examples/mysql/`** — end-to-end demo with schema, sqlc queries, generated
commands, and a runnable CLI.

### API stability note

The `runtime/` public API is **locked** for the `v0.x` series: minor versions
may add but not remove or rename existing symbols. A breaking change requires
a major version bump.

The generated file format (calls, import aliases) may change in minor versions
during `v0.x`. After any generator upgrade, run `go generate ./...` to
regenerate.
